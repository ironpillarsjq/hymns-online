## REMOVED Requirements

### Requirement: Discover PDF files from /public/data directory

**Reason**: PDF file discovery is replaced by image folder discovery (`image-data-source` spec). The application no longer supports PDF browsing.

**Migration**: Users SHALL replace `public/data/<site-title>/*.pdf` files with `public/data/<site-title>/<folder-name>/*.jpg` (or `.png`, `.webp`) image folders. The new `vite-plugin-image-manifest.js` discovers image folders and generates an updated manifest structure.

### Requirement: Build-time manifest generation

**Reason**: The manifest format changes from a flat PDF list to a folder-centric structure with image lists. The new manifest is generated by `image-data-source`.

**Migration**: The manifest JSON structure changes from `{ siteTitle, pdfs: [...] }` to `{ siteTitle, folders: [ { folderName, displayName, path, imageCount, images: [...] } ] }`. Consumers (TocPage, ViewerPage) SHALL be updated to read the new structure.

### Requirement: Fully static asset serving

**Reason**: This requirement remains valid but is now covered by `image-data-source` with image files instead of PDF files.

**Migration**: No migration needed for the concept itself — static serving still applies to image files. The `pdfjs-dist` web worker copy step in the Vite plugin SHALL be removed.
