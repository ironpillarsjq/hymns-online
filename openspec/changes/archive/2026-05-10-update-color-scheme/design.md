## 背景

当前应用使用暗色主题（深蓝紫色背景 `#1a1a2e`），与温暖、轻松的诗歌阅读氛围不符。用户反馈阅读时视觉偏暗，希望能更护眼、更明亮。

## 目标 / 非目标

**目标：**
- 将整体色调从暗色转为明亮的中性灰色调
- 保持一致的视觉层次（主色、次色、强调色、背景色、文字色）
- 确保所有组件的可访问性（对比度符合 WCAG 标准）
- 最小化代码改动，不影响现有功能

**非目标：**
- 不修改任何业务逻辑
- 不改变组件的结构和交互行为
- 不添加新组件或删除现有组件（仅调整颜色）

## 技术决策

### 1. 配色方案采用 CSS 变量

**选择**：使用 CSS 自定义属性（CSS Variables）集中定义颜色变量。

**理由**：
- 单一修改点，全局生效
- 便于后续主题切换（若未来需要暗色模式）
- 浏览器原生支持，无需额外依赖

**备选方案**：
- 硬编码颜色值：维护成本高，难以复用
- SCSS 变量：需要预处理器，额外构建步骤

### 2. 颜色层级定义

| 用途 | 颜色值 | 用途描述 |
|---|---|---|
| 主色（Primary） | `#626C83` | 按钮、链接、强调元素 |
| 次色（Secondary） | `#888FA0` | 次要按钮、边框、图标 |
| 浅色（Light） | `#D4D6DC` | 分割线、禁用状态 |
| 极浅（Very Light） | `#FAFAFA` | 卡片背景、悬浮效果 |
| 背景色（Background） | `#FFFFFF` | 主背景 |
| 文字色（Text） | `#797979` | 正文文字 |
| 强调色（Accent） | `#AEB3BE` | hover 状态、次要文字 |
| 错误色（Error） | `#e57373` | 错误提示（保留原色） |

### 3. 组件样式覆盖范围

在 `App.vue` 全局样式中定义 CSS 变量，在各组件样式中通过 `var(--color-*)` 引用：
- `App.vue`：全局背景、文字颜色
- `TocPage.vue`：列表项背景、hover 效果、标题颜色
- `PdfViewerPage.vue`：阅读器背景、加载状态
- `PageIndicator.vue`、`NavControls.vue`：浮层背景、按钮颜色

## 风险 / 取舍

- **[风险] 浅色背景下 PDF 阅读器对比度** → 保持阅读器内部为深色背景（`#0d0d1a`），仅外层容器使用浅色
- **[风险] 现有用户习惯** → 提供平滑过渡，无 abruptly 变化
- **[取舍] 白色背景可能比深色更刺眼** → 使用 `#FFFFFF` 作为页面背景，但 PDF 阅读器保持暗色，体验更佳
