## 新增需求

### 需求：全屏 PDF 渲染

系统应使用 PDF.js 在 `<canvas>` 元素上全屏渲染 PDF 页面，每次显示一页，填满整个视口。

#### 场景：初始加载显示首页
- **当** PDF 加载到阅读器中
- **则** 应渲染 PDF 的第一页，填满视口并保持长宽比

#### 场景：页面适配视口
- **当** 浏览器窗口尺寸改变
- **则** 当前显示的页面应重新渲染以适应新的视口尺寸，同时保持长宽比

### 需求：键盘导航

系统应支持键盘方向键翻页，模拟 PPT 幻灯片切换体验。

#### 场景：右箭头前进
- **当** 用户按下右箭头键且存在下一页
- **则** 阅读器应切换到下一页

#### 场景：左箭头后退
- **当** 用户按下左箭头键且存在上一页
- **则** 阅读器应切换到上一页

#### 场景：边界处理 - 首页
- **当** 用户在首页按下左箭头键
- **则** 不应发生任何操作（停留在第 1 页）

#### 场景：边界处理 - 末页
- **当** 用户在末页按下右箭头键
- **则** 不应发生任何操作（停留在最后一页）

### 需求：触控与滑动导航

系统应在手机和平板设备上支持触控滑动手势翻页。

#### 场景：左滑前进
- **当** 用户在触摸屏上向左滑动且存在下一页
- **则** 阅读器应切换到下一页

#### 场景：右滑后退
- **当** 用户在触摸屏上向右滑动且存在上一页
- **则** 阅读器应切换到上一页

### 需求：页码指示器浮层

系统应显示页码指示器（如"3 / 12"），以半透明浮层形式呈现，一段时间无操作后自动隐藏。

#### 场景：交互时显示页码
- **当** 用户导航到 20 页 PDF 的第 5 页
- **则** 浮层应显示"5 / 20"，3 秒无操作后淡出

#### 场景：操作后重新显示
- **当** 页码指示器已隐藏，用户按下导航键
- **则** 页码指示器应立即重新显示并更新为当前页码

### 需求：导航控件

系统应提供可见或易于发现的导航控件，包括上一页、下一页和"返回目录"按钮。

#### 场景：导航按钮显示
- **当** 用户鼠标悬停在视口底部附近（或在移动端点击屏幕）
- **则** 导航按钮（上一页、下一页、返回目录）应以浮层形式出现

#### 场景：点击下一页按钮
- **当** 用户点击"下一页"导航按钮
- **则** 阅读器应切换到下一页

#### 场景：返回目录
- **当** 用户点击"返回目录"按钮
- **则** 浏览器应导航回目录页面

### 需求：PDF 加载状态

系统应为 PDF 加载过程提供视觉反馈，并优雅处理加载错误。

#### 场景：加载进度
- **当** PDF 正在下载和解析中
- **则** 阅读器应显示加载动画或进度指示器

#### 场景：PDF 加载失败
- **当** PDF 加载失败（如文件未找到、网络错误）
- **则** 阅读器应显示错误信息，并包含"返回目录"链接
