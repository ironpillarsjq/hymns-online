## Context

当前项目是基于 Vanilla JS 的静态 SPA，无构建工具和服务器端运行时。已有 14 个 spec 覆盖目录浏览、图片阅读、导航栏、全屏切换等功能。本次变更新增帮助页面，同时调整目录页标题样式。

## Goals / Non-Goals

**Goals:**
- 目录页标题字体增大，与全屏按钮一同水平居中
- 导航栏新增"帮助"入口
- 新增帮助页面，从 `public/help.md` 加载内容并渲染为 HTML

**Non-Goals:**
- 不引入外部 Markdown 解析库
- 不改变全屏按钮的现有行为和定位方式
- 不改变导航栏其他链接的行为

## Decisions

### 帮助页面 Markdown 渲染

**决策**: 使用简单的客户端 Markdown-to-HTML 转换函数，而非引入第三方库。

**理由**: 
- 帮助页面内容简单（标题、段落、列表），不需要完整 Markdown 支持
- 避免增加外部依赖和包体积
- 函数约 50 行，覆盖 `#`/`##`/`###` 标题、粗体、列表、链接、换行

**备选**: 引入 `marked` 库 → 过于重量级；构建时预先生成 HTML → 增加构建复杂度。

### 路由实现

**决策**: 沿用项目现有 hash 路由方式（`#/help`），而非新增 HTML 文件。

**理由**: 与现有 `#/site/<name>`、`#/viewer` 路由方式一致，无需配置重定向。

### 标题居中实现

**决策**: 在 TOC 页面上方创建一个标题+全屏按钮的容器，使用 CSS flexbox 居中。

**理由**: 
- 全屏按钮已 `position: fixed` 水平居中，标题需与之视觉对齐
- 将标题也放在相同层级的居中容器中，使用 `text-align: center` 和适当间距即可

## Risks / Trade-offs

- [低风险] 自写 Markdown 渲染函数可能不支持复杂格式 → 缓解：帮助文档内容简单，够用即可
- [低风险] 标题居中可能在小屏幕上与导航栏重叠 → 缓解：使用适当 padding-top（约 40px）避开顶部导航栏
